CSS Transforms

The CSS3 Transform property lets you translate, rotate, scale, or skew any elements on the page. The value of the transform property is one or more transform functions separated by spaces and will be applied in the same order they are provided.

  • Translation function allows you to move elemnts by x from the left, and y from the top (alternatively you can use translatex(x) or translatey(y)): translate(x, y)
  • Scaling function scales the element by the defined factors horizontallyscale(x,y) if only one value is provided it is used for x and y, fore reducing the element to half: scale(.5)
  • The Rotation function rotates an element aroun the point of origin with positive degreesmoving clockwise, nevative can be used: rotate(10deg)
  • Skey function specifies a skey along the X and Y axes, if the second parameter is omited, the skew only ocurs on X axis: skew(10deg, 4deg)

To change the origin of the transform it has the same syntax as the background-position and defaults to the center of the object: tranform-origin: 0 0;

CSS Trasitions

Transitions allow to to change of the values of CSS properties over time providing animations. Likewise, you can animate any transforms. Here are the steps for a simple transition using only CSS

  1. Declare original state of the element in the default style declaration.
  2. Declare the final state of the transitioned element, for instance a hover state.
  3. Include the transition functions in default style declaration using few different properties:
    • transition-property: lists the CSS poperties that should be transitioned like: background, border and box model. transition-property: transforms
    • transition-duration: sets long the transition will take. it is specified in seconds: transition-duration: 0.2s
    • transition-timing-function: lets you control the pace of the transition in even more granular detail. Start off gast and end slower, or start off slow and get faster, key terms: ease, linear,ease-in, ease-out, or ease-in-out.
    • transition-delay: it is to introduce a delay before the animation begins, instead as immediately as usually.Default is zero: transition-delay: 200ms
    It is possible to specify muiltiple transitions when using shorthand transition property, specify all the calues for each transition together, selarate each transiotion with commas: transition: color 0.2s ease-out, transform 0.2s ease-out

Animations

Animations, unlike transitions, allow you to control eash steo of an animation via keyframes. The @keyframes would look like this

@keyframes 'appear'{ 0% { opacity: 0; } 100% { opacity: 1; } } @keyframes 'disappear'{ to { opacity: 0; } from { opacity: 1; } } @keyframes 'appearDisappear'{ 0%, 100% { opacity: 0; } 20%, 80% { opacity: 1; } }

  • animation-name: is used to attach an animation to an element: animation-name: 'appear';
  • animation-duration: this property defines the length of time, in seconds or miliseconds, an animation takes to complete one iteration (from 0% to 100%): animation-duration: 300ms;
  • animation-timing-function: determines how the animation will progress over its duration. The options are the same as for transition-timing-function: ease-in, ease-out, ease-in-out, or cubic-bezier. animation-timin-function: linear;
  • animation-iteration-count: this property lets ou define how many times the animation will play through, the value is generally an interger but it can also be decimal, if omited default is 1, infinite can be used: animation-iteration-count: infinite;
  • animation-direction: you can use the value alternate to make every other iteration play the animation backwards: animation-direction: alternate;
  • animation-delay: defines how many millicods or seconds to wait before the animation starts: animation-delay: 15s;
  • animation-fill-mode: defines what happens before the animation begins and after an animaiton ends. We can tell the animation to sit and wait on the first keyframe until the animation starts, or stop on the last keyframe without reverting to the original values ath teh conclusion of the animation, or both. Available values are:
    • forwards: the animcation continues to apply the values of the last keyframes after the animation ends.
    • backwards: animation's initial keyframes are applied as soon as the anymation style is applied to the element
    • both: appplies both, backwards and forwards, effects.
    The default value is none.
  • animation-play-state: defines whether the animation is running or paused. a pause animation displays the current state of the animation statically.

Shorthand animation property:

.verbose{ animation-name: 'appear'; animation-duration: 300mx; animation-timing-function: ease-in; animation-iteration-count: 1; animation-direction: alternate; animation-delay: 5s; animation-fill-mode: backwards; } /*shorthand*/ .concise{ animation: 'appear' 300 ms ease-in 1 alternate 5s backwards; } /*multiple animations*/ .concise{ animation: 'animationOne' 300 ms ease-in 1 alternate 5s backwards, 'animationTwo' 150 ms ease-out 3 alternate 5s forwards; }

Call backs are used to facilitate event-driven asynchronous programming. The increase in the use of asynchronous programming in JavaScript has meant that more and more callbacks are being used. This can result in messy and confusing “spaghetti code”.

A promise represents the future result of an asynchronous operation, they don’t do anything that can’t already be achieved using callbacks, but they help simplify the process, and avoid the convoluted code that can result from using multiple callbacks. When a promise is created, it calls an asynchronous operation and is then said to be pending, the promise is said to be unsettled. Once the operation has completed, the promise is said to have been settled. A settled promise can result in two different outcomes:

  • Resolved ― the asynchronous operation was completed successfully.
  • Rejected ― the asynchronous operation didn’t work as expected, wasn’t successfully completed or resulted in an error.
The general layout of a promise is

const promise = new Promise( (resolve, reject) => { // initialization code goes here if (success) { resolve(value); } else { reject(error); } } });

A pure function is a function that adheres to the following rules:

  • The return value of a pure function should only depend on the values provided as arguments. It doesn’t rely on values from somewhere else in the program
  • There are no side-effects. A pure function doesn’t change any values or data elsewhere in the program. It only makes non-destructive data transformations and returns new values, rather than altering any of the underlying data.
  • Referential transparency. Given the same arguments, a pure function will always return the same result.

Currying is a process that involves the partial application of functions.A function is said to be curried when not all arguments have been supplied to the function, so it returns another function that retains the arguments already provided, and expects the remaining arguments that were omitted when the original function was called. Currying relies on higher-order functions that are able to return partially applied functions

Ajax

Ajax is a technique that allows web pages to communicate asynchronously with a server, and it dynamically updates web pages without reloading.

The Fetch API provides a global fetch() method that only has one mandatory argument, which is the URL of the resource you wish to fetch.

fetch('https://example.com/data') .then( // code that handles the response ) .catch( // code that runs if the server returns an error ) fetch(url) .then((response) => { if(response.ok) { return response; } throw Error(response.statusText); }) .then( response => // do something with response ) .catch( error => console.log('There was an error!') )

Some other properties of the Response object are:

  • headers – A Headers object (see later section) containing any headers associated with the response
  • url – A string containing the URL of response
  • redirected – A boolean value that specifies if the response is the result of a redirect
  • type – A string value of “basic”, “cors”, “error” or “opaque”. A value of “basic” is used for a response from the same domain.

The response object also contains a number of methods that return promises that can then be chained together

The redirect() method can be used to redirect to another URL. It creates a new promise that resolves to the response from the redirected URL.
