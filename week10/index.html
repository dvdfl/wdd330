Validating Forms

When you enter data data in a web form, the browser and/or the web server will check to see that the data is in the correct format and within the constraints set by the application. Form Validation done by the browser is called client-side validation. There exists two different types of client-side validation:

  • Build-in - uses HTML from validation features, javascript requirements can be minimum. It may also be limited.
  • Javascript - Fulli customizable requires javascript coding or use of a library.

Built-in validations can be defined setting attributes on the form elements, some examples:

  • The required attribute specifies a form field needs to be filled in before the form can be submitted.
  • The minlength and maxlength attributes set the minimum and maximum length of the text value.
  • min and max attributes set the limits for a numeric integer values.
  • The type attribute defines what time of data to be entered, likea number or email address.
  • pattern attribute is to set a regular expression to define the pattern of the entered value

  • alert() function will stop program executio and display a provided message in a dialog box, this methods returns undefined. Example: alert("Hello world")
  • confirm() function also will stop program execution and display a confirmation dialog with a provided mesasge and the OK or cancel options, ir returns true if OK is selected or false when Cancel is chosenlet go = confirm("De you want to continue?");
  • The prompt(). function stops execution of the program and displays a message provided also an input field for the user to enter text. When the OK button is clicked it will return the value of the input as string, if cancel is clicked then null will be returned: let userInput = prompt("Please enter a your name:")

Browser informatioon can also be obtained through the window object window.navigator. And information about the URL can be obtained using window.location like protoco, hostname, path, port and search (query string).

Browser history can be also be accessed and navigted:

window.history.length; // shows how many pages have been visited window.history.go(1); // goes forward 1 page window.history.go(0); // reloads the current page window.history.go(-1); // goes back 1 page

Using Fetch

The Fetch API provides a global fetch() method that only has one mandatory argument, which is the URL of the resource you wish to fetch.

fetch('https://example.com/data') .then( // code that handles the response ) .catch( // code that runs if the server returns an error ) fetch(url) .then((response) => { if(response.ok) { return response; } throw Error(response.statusText); }) .then( response => // do something with response ) .catch( error => console.log('There was an error!') )

Some other properties of the Response object are:

  • headers – A Headers object (see later section) containing any headers associated with the response
  • url – A string containing the URL of response
  • redirected – A boolean value that specifies if the response is the result of a redirect
  • type – A string value of “basic”, “cors”, “error” or “opaque”. A value of “basic” is used for a response from the same domain.
